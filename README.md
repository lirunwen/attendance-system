# 考勤管理系统

> 给部门文员用的内网 Web 工具，处理员工考勤打卡数据的导入、统计和异常管理。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| **用户登录** | 账号密码登录，默认管理员 admin / admin123，支持注册新用户 |
| **班次管理** | 定义上下班时间、迟到/早退宽容分钟数（如白班 09:00-18:00） |
| **地点管理** | 定义打卡地点及匹配关键词（如"主楼""北门"），导入时自动识别 |
| **员工管理** | 维护员工信息（工号、姓名、默认班次），支持 Excel 批量导入 |
| **数据导入** | 下载标准 Excel 模板，填入打卡记录后上传，系统自动统计 |
| **考勤查看** | 按月日历视图展示所有人出勤情况，颜色标记迟到/早退/缺卡 |
| **异常反馈** | 列出所有异常记录，文员可填写处理意见 |
| **统计报表** | 月度出勤率图表、迟到排行、异常分布 |
| **数据导出** | 考勤/异常/统计均可导出 Excel，**支持自定义选择导出列** |

## 快速开始

### 环境要求

- Python 3.8+
- 办公网内任一 Windows / Mac / Linux 电脑

### 启动步骤

```bash
# 1. 进入项目目录
cd attendance

# 2. 创建虚拟环境（仅首次）
python3 -m venv venv
source venv/bin/activate        # Mac / Linux
# 或 venv\Scripts\activate      # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 启动服务
python app.py
```

终端显示如下即启动成功：

```
 * Running on http://127.0.0.1:5100
 * Running on http://192.168.x.x:5100
```

### 访问方式

- **本机**：浏览器打开 `http://127.0.0.1:5100`
- **同局域网其他电脑**：浏览器打开 `http://你电脑的IP:5100`

> 如果端口 5100 被占用，编辑 `app.py` 修改 `port` 值。

## 使用流程

### 第 1 步：基础配置

1. **添加班次** → 如白班 09:00-18:00，迟到宽容 15 分钟
2. **添加地点** → 如"主楼"，关键词填"主楼,办公楼"
3. **添加员工** → 逐一添加或下载模板批量导入

### 第 2 步：导入打卡数据

1. 在**数据导入**页面下载 Excel 模板
2. 按模板格式填入打卡记录（姓名、工号、日期、打卡时间、打卡地点）
3. 上传 Excel，系统自动：
   - 按关键词匹配打卡地点
   - 将每日首条打卡记为上班卡、末条记为下班卡
   - 判定迟到/早退/缺卡状态

### 第 3 步：查看与处理

- **考勤查看** → 月度总览，颜色一目了然
- **异常反馈** → 处理迟到、缺卡等异常，填写备注
- **统计报表** → 月度数据图表和排行

## 考勤判定规则

| 条件 | 判定结果 |
|------|---------|
| 上班打卡 ≤ 班次上班时间 + 宽容分钟数 | 正常 |
| 上班打卡 > 班次上班时间 + 宽容分钟数 | 迟到 |
| 下班打卡 < 班次下班时间 - 宽容分钟数 | 早退 |
| 当天仅有 1 条打卡记录 | 缺少下班卡 |
| 当天无打卡记录 | 缺勤 |

## 常见问题

**Q：数据存在哪里？**
A：项目 `instance/` 目录下的 `attendance.db` 文件，无需安装数据库。

**Q：数据会丢失吗？**
A：`attendance.db` 是持久化文件，只要不删除就不会丢失。建议定期备份。

**Q：如何更换端口？**
A：修改 `app.py` 最后一行 `port` 参数，重启即可。

**Q：如何更新员工信息？**
A：在员工管理页面编辑，或禁用离职员工（保留考勤历史）。
